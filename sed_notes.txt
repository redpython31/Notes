The -r option in sed enables extended regular expressions (ERE) mode. This is different from the default basic regular expressions (BRE) mode, which is more limited in the types of patterns you can use.

Here's what happens:
In basic regular expressions (BRE), certain special characters (like + for "one or more") are treated as literal characters unless you escape them with a backslash (\).

For example, in basic mode:

sed 's/[0-9]\+/digit/' file
Here, \+ is used to match one or more digits.

However, in extended regular expressions (with the -r option), you don't need to escape the + symbol. So the same command can be written as:

sed -r 's/[0-9]+/digit/' file
This makes the regular expression more readable and easier to write since you don't need to escape certain characters.

To summarize:
Without -r, + in [0-9]\+ needs escaping (for BRE).

With -r, + works directly as "one or more" (for ERE).

So, the command sed -r 's/[0-9]+/digit/' file will replace any sequence of one or more digits ([0-9]+) with the word "digit" in the file.
